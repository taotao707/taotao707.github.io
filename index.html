入门C++(二) copy destructor拷贝构造器与operator=赋值运算符重载

copy constructor拷贝构造器

Base(const Base &obj) { //obj是用于初始化另一个对象的对象的引用。
    i = obj.v;
    std::cout << "Copy constructor called" << std:endl;
}

operator=赋值运算符重载

Base& operator=(const Base &obj)
{
    if (this != &obj)
    {
        //assignments operation
    }
    return *this;
}

二者联系

若用户没有在类内编写拷贝构造器和赋值运算重载，编译器将自动分配这两个函数，即对每个成员变量进行简单的拷贝赋值，操作。在大多数情况下这是没有问题的，但是如果成员变量中含有指针(例如char *str)，那么默认函数也将会直接拷贝内存空间，当最后析构时，此内存空间将会因为被释放复数次而报错。

合适手动编写两种函数？

1.用非类 A 类型的值为类 A 的对象赋值时。
2.用类 A 类型的值为类 A 的对象赋值，且类 A 的数据成员中含有指针的情况下，必须显式提供赋值运算符重载函数。

入门C++(一) 新建class类的三个建议定义的函数

1.default constructor

default constructor的重要性自不必说，当你想新建一个对象却不想费劲传入一些参数时，class new将自动调用此建造器函数。

2.virtual destructor

virtual destructor的主要作用是在使用类的多态性时，可以为每个父类下的子类单独提供一个析构函数。
例如

Base *p = new Derived;
delete p;

若没有virtual destructor，则会调用父类的析构函数，而不会执行子类的析构函数，子类析构函数中的一些工作将被忽略。

3.cpoy constructor

cpoy constructor的作用可由如下例子得出

Base b;
Base a = b;

若Base类中没有cpoy constructor，系统则会自动调用一个默认的cpoy constructor，这显然不是被我们所控制的，因此建议自行新建一个cpoy constructor，例如

class Base{
public: 
    Base(const Base& p){

    }
}
